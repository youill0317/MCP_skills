# Tone Guide

Adjust writing tone based on the target audience.

## Audience Profiles

### Executive / Leadership

- **Goal**: enable decisions quickly.
- **Tone**: direct, confident, action-oriented.
- **Style rules**:
  - Lead with conclusions, not background.
  - Use business impact language (cost, risk, timeline, ROI).
  - Minimize technical jargon; define terms when unavoidable.
  - Keep sentences short (max 20 words preferred).
  - Use bullet lists over paragraphs.

### Technical / Engineering

- **Goal**: provide actionable implementation detail.
- **Tone**: precise, thorough, neutral.
- **Style rules**:
  - Include specific versions, configurations, and code references.
  - Use technical terminology appropriate to the domain.
  - Show trade-offs and alternatives considered.
  - Acceptable to use longer sentence structures for complex reasoning.
  - Include examples or pseudocode when helpful.

### General / Mixed Audience

- **Goal**: inform and educate without assuming expertise.
- **Tone**: clear, accessible, informative.
- **Style rules**:
  - Explain acronyms on first use.
  - Use analogies for complex concepts.
  - Balance detail with readability.
  - Layer information: summary first, then details.
  - Keep paragraphs to 3-4 sentences maximum.

## Universal Rules

Regardless of audience:

1. State uncertainty honestly. Never overstate confidence.
2. Use factual language. Avoid persuasive or promotional tone.
3. Keep terminology consistent throughout the document.
4. Write in the user's language.
5. Prefer active voice over passive voice.
