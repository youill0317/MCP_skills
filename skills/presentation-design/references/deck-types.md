# Deck Types

Use this guide to select structure, pacing, and slide count by presentation intent.

## 1) Pitch Deck

- Primary goal: secure buy-in, funding, or partnership.
- Typical audience: investors, executives, strategic partners.
- Suggested length: 10-12 slides.
- Default section order:
  1. Hook and problem
  2. Target user or market
  3. Solution
  4. Evidence or traction
  5. Business model or value logic
  6. Go-to-market
  7. Team or capability
  8. Ask and next step

## 2) Executive Update

- Primary goal: decision support and alignment.
- Typical audience: leadership or steering committee.
- Suggested length: 6-10 slides.
- Default section order:
  1. Decision summary
  2. Current state
  3. Key metrics and changes
  4. Risks and blockers
  5. Recommendation
  6. Required decision

## 3) Research Talk

- Primary goal: communicate method, results, and implications credibly.
- Typical audience: academic, technical, or expert audiences.
- Suggested length: 12-20 slides.
- Default section order:
  1. Research question
  2. Background and gap
  3. Method
  4. Results
  5. Interpretation
  6. Limitations
  7. Implications and next work

## 4) Education Lecture

- Primary goal: teach clearly and improve retention.
- Typical audience: students, trainees, workshop attendees.
- Suggested length: 12-30 slides (duration-dependent).
- Default section order:
  1. Learning objectives
  2. Concept explanation
  3. Worked examples
  4. Practice or checkpoint
  5. Summary and recall prompts

## 5) Product Demo

- Primary goal: prove product value through scenario-based walkthrough.
- Typical audience: customers, internal stakeholders, evaluators.
- Suggested length: 8-14 slides.
- Default section order:
  1. User problem context
  2. Demo scenario
  3. Key capabilities
  4. Differentiation
  5. Adoption and next step

## 6) Proposal Deck

- Primary goal: get approval for a concrete plan.
- Typical audience: decision makers, sponsors, clients.
- Suggested length: 8-15 slides.
- Default section order:
  1. Objective and scope
  2. Approach options
  3. Recommended plan
  4. Timeline and resources
  5. Risks and mitigation
  6. Approval request

## Type Selection Rules

1. If the user asks for investment or sponsorship, prefer Pitch Deck.
2. If the user asks for a decision with status context, prefer Executive Update.
3. If method and evidence rigor are central, prefer Research Talk.
4. If learning outcomes are explicit, prefer Education Lecture.
5. If live capability proof is central, prefer Product Demo.
6. If approval of a plan is central, prefer Proposal Deck.
