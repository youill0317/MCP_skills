# Data Visualization Rules

Choose chart forms based on the analytical question, then verify integrity.

## Chart Selection by Intent

1. Comparison across categories -> bar chart or dot plot.
2. Trend over time -> line chart.
3. Part-to-whole composition -> stacked bar (small category count) or 100% stacked bar.
4. Distribution -> histogram or box plot.
5. Relationship between variables -> scatter plot.

Use tables when exact values matter more than pattern recognition.

## Evidence Framing

1. State metric definition and unit near the visual.
2. Add time window and population scope for all comparative claims.
3. Mark data source or origin in notes when provided.
4. Flag missing context with `[EVIDENCE NEEDED]`.

## Integrity Checks

1. Do not truncate axes in a way that distorts interpretation unless explicitly justified.
2. Do not use 3D charts for quantitative comparison.
3. Keep consistent scales when comparing related visuals.
4. Avoid dual-axis charts unless both scales are clearly interpretable.
5. Separate observed data from forecast or assumption visually.

## Annotation Rules

1. Annotate only the point that supports the slide message.
2. Keep annotations short and decision-relevant.
3. Use one highlight color for emphasis.

## Common Failure Modes

1. Chart type chosen by aesthetics rather than question type.
2. Too many series on one chart for the available viewing time.
3. Missing denominator or baseline for percentage claims.
4. Conclusion text stronger than the visual evidence allows.
