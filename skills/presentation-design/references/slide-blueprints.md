# Slide Blueprints

Use these role-based blueprints to keep slides intentional and easy to map to any tool.

## Blueprint Schema

Each slide spec should include:

1. Slide role
2. Slide title
3. One key message sentence
4. Content blocks (2-5 blocks)
5. Visual blueprint
6. Data requirements (if needed)
7. Speaker notes

## Common Slide Roles

## 1) Hook

- Purpose: earn attention and establish relevance.
- Content blocks: audience problem, consequence, urgency.
- Visual blueprint: bold headline + one anchoring visual.
- Avoid: generic mission statements without tension.

## 2) Context

- Purpose: align on baseline assumptions and scope.
- Content blocks: current state, boundaries, constraints.
- Visual blueprint: concise bullets + simple framing diagram.
- Avoid: dense history timeline with no present relevance.

## 3) Problem

- Purpose: define the core issue precisely.
- Content blocks: symptoms, root cause candidates, impact.
- Visual blueprint: issue tree, impact matrix, or before/after.
- Avoid: listing issues without prioritization.

## 4) Insight

- Purpose: surface the non-obvious takeaway.
- Content blocks: observation, interpretation, implication.
- Visual blueprint: highlighted metric + short interpretation callout.
- Avoid: raw data dump with no narrative.

## 5) Evidence

- Purpose: substantiate the key claim.
- Content blocks: source context, metric/result, confidence note.
- Visual blueprint: single chart with direct annotations.
- Avoid: multiple chart types competing on one slide.

## 6) Option Comparison

- Purpose: support choice between alternatives.
- Content blocks: options, evaluation criteria, trade-offs.
- Visual blueprint: side-by-side matrix or weighted score table.
- Avoid: conclusion without transparent criteria.

## 7) Plan or Roadmap

- Purpose: show execution path.
- Content blocks: phases, milestones, owners, dependencies.
- Visual blueprint: horizontal timeline or phase swimlane.
- Avoid: dates without deliverables.

## 8) Ask or Action

- Purpose: make the next decision explicit.
- Content blocks: requested decision, rationale, deadline.
- Visual blueprint: short decision card with 2-3 actions.
- Avoid: vague "thank you" close without action.

## Slide Hygiene Rules

1. Keep one dominant message per slide.
2. Keep text concise and scannable.
3. Do not mix more than one primary visualization type on a slide.
4. State assumptions in speaker notes when evidence is incomplete.
