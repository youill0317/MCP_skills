# Problem Types

Use this guide to choose the primary problem framing mode.

## 1) Project/Product Problem

- Core intent: improve outcomes in delivery, product behavior, process, or operations.
- Typical inputs: KPI drops, user complaints, delays, quality issues, adoption problems.
- Primary unit of analysis: system behavior and execution constraints.
- Common framing axis:
  1. Who is affected?
  2. What is failing relative to target?
  3. Where in the flow does failure appear?
  4. Since when and under what conditions?

## 2) Research Problem

- Core intent: clarify a testable knowledge question and reduce ambiguity in causal interpretation.
- Typical inputs: hypothesis uncertainty, conflicting findings, unclear variable definitions.
- Primary unit of analysis: constructs, variables, and evidence quality.
- Common framing axis:
  1. What is the precise research question?
  2. Which variables/constructs are central?
  3. What competing explanations are plausible?
  4. What evidence would support or falsify each explanation?

## Selection Rules

1. If the user requests operational improvement or execution decisions, choose Project/Product.
2. If the user requests hypothesis clarity or causal/evidential interpretation, choose Research.
3. If signals are mixed, ask for type confirmation before full analysis.

## Failure Modes by Type

### Project/Product

1. Defining the problem as a broad business goal instead of an observable gap.
2. Treating downstream symptoms as root causes.
3. Ignoring stakeholder or dependency constraints.

### Research

1. Vague constructs without operational definitions.
2. Causal claims not distinguishable from correlation.
3. Missing alternative explanations or confounders.
