# Validation Questions

Generate questions that can disconfirm weak assumptions and sharpen problem framing.

## Question Design Rules

1. Prefer questions that reduce decision uncertainty.
2. Make each question answerable with observable evidence.
3. Keep one uncertainty target per question.
4. Prioritize questions that can falsify dominant assumptions.

## Question Categories

1. Definition clarity
   - Are key terms operationally defined and stable?
2. Scope validity
   - Is the stated problem consistent across segments and contexts?
3. Causal plausibility
   - What competing cause explains the same symptom?
4. Data quality
   - Could missing or biased measurement explain the gap?
5. Decision relevance
   - If answered, would this change the problem framing materially?

## Prioritization Heuristic

Score each question on:

1. `Impact`: how much framing confidence could improve
2. `Feasibility`: how quickly evidence can be collected
3. `Risk`: downside of being wrong if unanswered

Address highest combined score first.

## Output Format

For each question, include:

1. question text
2. uncertainty addressed
3. evidence needed
4. expected decision impact
5. priority (high/medium/low)
