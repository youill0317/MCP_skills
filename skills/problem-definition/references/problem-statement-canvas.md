# Problem Statement Canvas

Use this canvas to convert vague concerns into a testable problem statement.

## Required Fields

1. `Context`: where the issue occurs and for whom.
2. `Current State`: what is observed now.
3. `Desired State`: what should be true instead.
4. `Gap`: measurable difference between current and desired states.
5. `Impact`: consequence if the gap remains unresolved.
6. `Time Boundary`: when the issue started or when it must be resolved.
7. `Constraints`: limits that shape valid interpretations.

## One-Sentence Problem Formula

`For [affected entity], [current state] differs from [desired state] by [gap], causing [impact] within [time boundary] under [constraints].`

## Scope Boundary Rules

1. Define in-scope elements that must be explained.
2. Define out-of-scope elements to prevent overreach.
3. Separate objective constraints from assumptions.

## Signal Definition Rules

1. Define at least one success signal and one failure signal.
2. Prefer measurable signals over subjective wording.
3. If a signal cannot be measured yet, mark `[INPUT NEEDED: metric definition]`.

## Example (Project/Product)

`For new users in onboarding, day-7 activation is 24% versus a 40% target, reducing trial-to-paid conversion by 12 points this quarter under unchanged acquisition mix and support capacity.`

## Example (Research)

`For adult learners in remote cohorts, observed retention differs across feedback frequency conditions, but the direction and magnitude are unclear because baseline motivation and prior experience are not yet controlled.`
