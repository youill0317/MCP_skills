# Workflow and Output Contract

Follow this execution contract for consistent outputs.

## Inputs to Collect

1. Topic domain and research question (or thesis).
2. Paper type.
3. Required length and deadline.
4. Mandatory sections from class or journal guidance.
5. Available source list and preferred citation density.

If any required input is missing, continue with explicit assumptions and list them at the top.

## Workflow

1. Generate a section-level outline.
2. Draft section by section.
3. Attach in-text citations while drafting, not after.
4. Compile and normalize reference list.
5. Run quality gate and produce issue log.

## Output Contract

Return output in this order:

1. `Metadata`: title, paper type, language, assumptions.
2. `Outline`: heading tree.
3. `Manuscript Draft`: full text by section.
4. `References (APA 7)`.
5. `Tables/Figures` (if any).
6. `Compliance Report`: pass/fail checks and fix tasks.

## Issue Reporting

Use explicit tags:

- `[CITATION NEEDED]` for unsupported claims.
- `[MISSING METADATA: field]` for incomplete references.
- `[VERIFY SOURCE]` for questionable source reliability.
