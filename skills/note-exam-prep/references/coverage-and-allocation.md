# Coverage And Allocation

Load this reference when the user requires complete note coverage with no meaningful omissions.

## Coverage Map

Before drafting questions, list every testable source item:

1. Section titles and subsection boundaries.
2. Definitions and key terms.
3. Claims, principles, and rules.
4. Ordered processes or stages.
5. Comparisons, contrasts, and exception cases.
6. Tables, formulas, diagrams described in text, and code examples.
7. Examples or case fragments that clarify the rule.

Each item must be represented by at least one question.

## Allocation Rules

- Prefer multiple-choice for comparisons, exceptions, code reading, and concept discrimination.
- Prefer fill-in-the-blank for exact terms, formulas, ordered labels, and short recall targets.
- Do not force equal counts. Allocate by note content.
- If one section is dense, assign more questions to it.
- If two adjacent items are inseparable, one question may cover both, but only if the explanation names both.

## Ordering Rules

- Follow the note's heading order.
- Keep related subtopic questions together.
- Do not move advanced details ahead of prerequisite concepts unless the note itself does.
