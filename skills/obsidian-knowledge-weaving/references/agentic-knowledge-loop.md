# Agentic Knowledge Loop for Obsidian

Use this loop every session to accumulate reusable knowledge.

## Loop

1. **Capture**
   - Write the new idea, finding, or decision quickly.
   - Add at least one explicit link to an existing note.
2. **Contextualize**
   - Search for 3-5 related notes.
   - Read only relevant sections.
   - Attach 2-4 high-value links (concept, source, project, hub).
3. **Consolidate**
   - Move durable insights into canonical concept notes.
   - Keep ephemeral details in project/source notes.
4. **Index**
   - Update one MOC/hub note with new or changed nodes.
5. **Queue**
   - Record unresolved questions and "next search" cues.

## Session Checklist (Operational)

- Did this note gain at least one inbound path via hub or related note?
- Is there a canonical concept note that should absorb part of this content?
- Are weak links removed or replaced with semantic links?
- Is at least one future retrieval query recorded?

## Lightweight Link Taxonomy

Use headings or inline markers consistently:

- **Related**: adjacent concepts
- **Depends on**: prerequisites
- **Used by**: applications/projects
- **Evidence**: source-backed support
- **Open loop**: unresolved item

## Retrieval Query Patterns

When returning later, search with combinations of:

- concept keyword + project name,
- alias + tag,
- open question phrase,
- source author/topic + decision keyword.

## Quality Gate

A note update is complete only if all are true:

1. At least one meaningful outbound link exists.
2. At least one discoverability path exists from a hub/MOC or neighboring concept.
3. A short summary line exists near the top.
4. A next-step retrieval cue exists.
